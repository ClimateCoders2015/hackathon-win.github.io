Our Solution

By using the geolocation function on Java script, we have accurately obtained the latitude and the longitude of the user, after that we use the Haversine formula to calculate the displacements between the location and the food banks. The code then automatically selects the closest foodbank. While this is not as accurate as calculating distance, it does give a representation of the closeness of a foodbank to that location. While we did try to use an API key for an accurate distance calculator, the Google Maps API was $200, which was way too high. We worked around this problem by adding possible deviation values and conditions. We used javascript to execute a cost calculator based on the data to help with estimation. The cost calculator takes the distance that is the displacement multiplied by 2 which was calculated for the closest food bank, and then divides it by the km per liter and then multiplies it with the months fuel price (2.92 for special 95 during the month of march). It also adds any additional costs that may be associated with the delivery. We have also included a system that provides directions using arrays.

To combat these twin terrors, we must embrace holistic solutions that navigate the intricate web of environmental, social, and economic factors at play.
